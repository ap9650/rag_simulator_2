Understand RAG Visually

This simulator explains how Retrieval-Augmented Generation (RAG) works step-by-step. Think of RAG as giving an LLM access to an external brain of documents before generating answers. Click through each stage to understand the entire pipeline visually.

RAG Workflow

1. Documents

Raw PDFs, websites, notes, manuals, etc.

2. Chunking

Breaking large documents into smaller pieces.

3. Embeddings

Converting text into vectors/numbers.

4. Vector Database

Storing semantic meaning for retrieval.

5. User Query

User asks a question.

6. Similarity Search

Finding the most relevant chunks.

7. LLM Generation

LLM uses retrieved context to answer.
